# ゲーム企画書：Article Generator

## 1. 概要
本作は『クッキークリッカー』をベースとしたクリッカーゲームである。
記事を読まずにコンテンツを消費する現代の読者に対するアンチテーゼとして、仏教観（輪廻転生、空、業、自我）の現代的解釈をゲーム体験を通じて提示する。

## 2. ゲームシステム

### 2.1 基本サイクル
1. **記事の生成**: 画面中央のキーボードとモニタをクリックして「記事」を生成。
2. **PV数**: 画面上部に表示。4と3の間をランダムに推移し続ける（不変の虚無）。
3. **施設の購入**: 蓄積した「記事」を消費して施設を開放し、自動生成（CPS）を強化する。

### 2.2 施設一覧
下表の順に高レベル・高効率な施設となる。

| 施設名 | 解説 |
| :--- | :--- |
| 人力（手動） | デフォルト。無強化状態で1クリックにつき1記事を生成。 |
| ゴーストライター | 低賃金で雇われた学生。 |
| コピペ | Stack Overflow等からの無断転載。 |
| LLM | 内容は破綻しているが生成速度は速い。 |
| 自動執筆パイプライン | git pushに連動して記事を自動生成する。 |
| AGI | 汎用人工知能。人間による執筆を不要にする。 |
| 虚空からの抽出 | アカシックレコードへのハッキングによる情報取得。 |
| 猿のタイプライター | 無限の猿がシェイクスピアを書き上げる確率論的執筆。 |
| 情報の広義積分 | 情報を無限大に発散させ、宇宙を満たす。 |

### 2.3 悟りと転生（周回要素）
*   **転生の条件**: 累計「記事」数が100兆に達すると、「悟りに達しましたか？」という問いが表示される。
*   **業（カルマ）システム**: 初回の「悟り」により解放。2周目以降、特定の条件を満たすことで「得」を入手可能。
*   **得（徳）の用途**: 「有漏（うろ）の得」として、基礎CPSの向上などに使用（本家におけるヘブンリーチップス枠）。

## 3. 「得」の獲得方法案
「得」の入手方法については、以下のいずれかの実装を検討する。

*   **案A（CUI）**: コンソールウィンドウで「南無阿弥陀仏」とタイピングする。
*   **案B（GUI）**: 専用の入力フォームを設け、そこに「南無阿弥陀仏」と入力する。
*   **案C（禅）**: 浄土教的なアプローチ（念仏）ではなく、禅宗的な要素（座禅/放置など）を取り入れた別システム。

## 4. 補足：計算式
施設の購入コスト算出には以下の計算式を用いる。

$$Cost = BaseCost \times 1.15^{NumberOfBuildings}$$
